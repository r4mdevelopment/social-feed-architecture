### Описание

Данный проект моделирует процесс обработки публикаций в социальной сети на уровне архитектуры СМО (системы массового обслуживания).

**Основные компоненты:**
- Источник заявок (пользователи/журналисты);
- Буфер (очередь публикаций);
- Приборы (серверы обработки);
- Диспетчер постановки и диспетчер выбора, реализующие дисциплины:
  - D1031 — заполнение буфера по кольцу;
  - D10O3 — выбивание самой старой заявки при переполнении;
  - D2B2 — выборка из буфера по принципу LIFO;
  - D2P2 — циклический выбор прибора;
  - И32 — равномерный закон межприбытия;
  - П31 — экспоненциальное время обслуживания.

---

## Программа

Данная программа (Python) реализована в пошаговом режиме с графическим интерфейсом (Tkinter).

### Возможности программы

- **Календарь событий** в табличной форме.
- Пошаговая обработка событий:
  - кнопка **«Следующий шаг»** — обработка одного события календаря;
  - кнопка **«Запустить N шагов»** — выполнение сразу N событий.
- Ключевые параметры календаря событий:
  - **cursor (D2P2)** — указатель, показывающий, какой прибор будет выбран следующим по циклу;
  - **last_index (D1031)** — индекс последней вставки в кольцевой буфер;
  - **% отказов** — процент выбитых заявок по дисциплине D10O3.
- **Ручная** настройка параметров (уже предустановленных).
- Подробный текстовый лог.

---

## Типы событий в календаре

### **ARRIVAL** — *генерация новой заявки*  
Источник (пользователь) создаёт новую публикацию.

### **BUFFER_ENQUEUE** — *постановка заявки в буфер (D1031)*  
Используется дисциплина заполнения по кольцу — D1031.

### **BUFFER_EVICT** — *выбивание заявки (D10O3)*  
Буфер заполнен → выбивается самая старая заявка; увеличивается статистика отказов.

### **BUFFER_PICK** — *выборка заявки из буфера (D2B2)*  
Извлечение последней по времени заявки (LIFO).

### **PACKET_FORMED** — *формирование пакета заявок одного источника*  
После первой выборки формируется пакет заявок только от одного источника.
Другие заявки возвращаются в буфер.

### **ASSIGN_TO_DEVICE** — *прямая постановка на прибор (опционально)*  
Заявка может миновать буфер (по умолчанию выключено).

### **SERVICE_START** — *передача заявки на прибор (D2P2)*  
Заявка направлена на обработку в прибор, выбранный циклически.  
cursor показывает следующий прибор, который будет выбран по кругу.

### **SERVICE_COMPLETE** — *завершение обработки*  
Прибор завершил работу с заявкой и освободился.
